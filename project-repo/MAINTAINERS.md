## Maintainer Responsibilities

Maintainers are expected to:

- Review and merge pull requests in a timely manner
- Triage issues and respond to bug reports and feature requests
- Participate in project discussions and decision-making
- Uphold the project's Code of Conduct
- Attend TSC meetings and maintainer syncs when possible
- Mentor contributors and help grow the community
- Maintain documentation and ensure it stays current
- Ensure the project follows security best practices

## Becoming a Maintainer

Contributors may be nominated to become maintainers based on the following criteria:

- Sustained, high-quality contributions to the project over a period of time
- Demonstrated understanding of the project's codebase, goals, and community
- Active participation in code reviews and issue discussions
- Commitment to the project's Code of Conduct and governance

### Nomination Process

1. An existing maintainer nominates a contributor by opening a pull request to add them to this file.
2. The nomination is discussed among current maintainers.
3. A vote is conducted in accordance with the project's [GOVERNANCE.md](GOVERNANCE.md).
4. If approved by a quorum of maintainers, the nominee is added to this file and granted write access to the repository.

## Stepping Down

Maintainers may step down at any time by opening a pull request to remove themselves from this file. We appreciate all contributions and understand that circumstances change.
